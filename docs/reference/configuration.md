# Configuration Reference

Configuration for `MicrosoftOutlookModule`, supplied through `forRoot()` or `forRootAsync()`.

## Registration methods

| Method | Signature | Notes |
|--------|-----------|-------|
| `forRoot` | `MicrosoftOutlookModule.forRoot(config: MicrosoftOutlookConfig)` | Synchronous registration with a static config object. |
| `forRootAsync` | `MicrosoftOutlookModule.forRootAsync({ useFactory, inject, imports })` | Async registration; `useFactory` returns `MicrosoftOutlookConfig` (or a Promise of it). |

Both are generated by `ConfigurableModuleBuilder`. The module registers `ScheduleModule`,
`EventEmitterModule`, and the required TypeORM entities internally.

## `MicrosoftOutlookConfig`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `clientId` | `string` | Yes | — | Microsoft Entra application (client) ID. |
| `clientSecret` | `string` | Yes | — | Microsoft Entra client secret. |
| `redirectPath` | `string` | Yes | — | OAuth callback path, e.g. `auth/microsoft/callback`. |
| `backendBaseUrl` | `string` | Yes | — | Public base URL of the backend, e.g. `https://api.example.com`. Used to build the redirect URI and webhook notification URLs. |
| `basePath` | `string` | No | — | API base path prepended to controller routes, e.g. `api/v1`. |
| `calendarWebhookPath` | `string` | No | `/calendar/webhook` | Path of the calendar webhook endpoint. |
| `state` | `MicrosoftOutlookStateConfig` | No | in-memory | Shared-state backend for locks and rate-limit budgets across containers. |
| `revocationEmitFlagTtlMs` | `number` | No | `604800000` (7 days) | TTL for the dedupe flag that suppresses duplicate `USER_REFRESH_TOKEN_INVALID` emits during webhook bursts. |
| `appOnly` | `AppOnlyAuthConfig` | No | — | Configuration for app-only (client credentials) authentication. Enables tenant-wide access. |

## `MicrosoftOutlookStateConfig`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `redis.client` | `RedisLike` | Yes (within `redis`) | — | Host-provided ioredis-compatible client. The module never imports ioredis itself. |
| `redis.keyPrefix` | `string` | No | `"outlook:"` | Prefix applied to all Redis keys. |
| `redis.required` | `boolean` | No | `false` | If `true`, module init throws when the Redis `PING` probe fails (instead of falling back to in-memory). |

When `state.redis.client` is omitted, the lock store and rate-limit store use in-process
in-memory implementations, suitable only for single-container deployments.

## `AppOnlyAuthConfig`

Configuration for app-only (client credentials) authentication. When enabled, the module
can access calendars and emails for all users in a Microsoft 365 tenant without individual
user consent. See [Connect an Enterprise Tenant](../how-to/connect-enterprise-tenant.md)
for usage guide and [DR-006](../decision-records/dr-006-dual-auth-architecture.md) for
design rationale.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `enabled` | `boolean` | Yes | — | Enable app-only authentication mode. When `true`, tenant-scoped services become available. |
| `tenantId` | `string` | Yes | — | Azure AD tenant ID (GUID format). Required because app-only auth targets a specific tenant. |
| `scopes` | `string[]` | No | `['https://graph.microsoft.com/.default']` | Microsoft Graph API scopes. Default requests all statically configured app permissions. |
| `tokenCacheTtlMs` | `number` | No | `3300000` (55 min) | TTL for cached app-only tokens. Set slightly below token lifetime to refresh before expiry. |
| `certificate` | `CertificateAuthConfig` | No | — | Certificate authentication (more secure). Takes precedence over `clientSecret`. |

## `CertificateAuthConfig`

When certificate authentication is used (recommended for production), provide the certificate
thumbprint and private key. See [DR-007](../decision-records/dr-007-certificate-authentication.md)
for security considerations.

The private key can be provided in three ways (priority: `privateKey` > `privateKeyPath` > `privateKeyBase64`):

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `thumbprint` | `string` | Yes | — | Certificate thumbprint (SHA-256 hash, base64url-encoded, 43 chars). From Azure AD portal or computed from cert. |
| `privateKey` | `string` | No | — | PEM-encoded private key as direct string. |
| `privateKeyPath` | `string` | No | — | Path to PEM file. Read at service initialization. |
| `privateKeyBase64` | `string` | No | — | Base64-encoded PEM key. Useful for environment variables. |
| `certificatePath` | `string` | No | — | Path to certificate PEM file (optional, for auto-computing thumbprint). |
| `certificateBase64` | `string` | No | — | Base64-encoded certificate PEM (optional, for auto-computing thumbprint). |

**Example with certificate (direct key):**
```typescript
MicrosoftOutlookModule.forRoot({
  clientId: process.env.AZURE_CLIENT_ID,
  clientSecret: '', // Not used with certificate
  redirectPath: 'auth/microsoft/callback',
  backendBaseUrl: process.env.BACKEND_URL,
  appOnly: {
    enabled: true,
    tenantId: process.env.AZURE_TENANT_ID,
    certificate: {
      thumbprint: process.env.AZURE_CERT_THUMBPRINT,
      privateKey: process.env.AZURE_CERT_PRIVATE_KEY,
    },
  },
})
```

**Example with certificate (file path):**
```typescript
appOnly: {
  enabled: true,
  tenantId: process.env.AZURE_TENANT_ID,
  certificate: {
    thumbprint: process.env.AZURE_CERT_THUMBPRINT,
    privateKeyPath: '/secrets/private-key.pem',
  },
}
```
