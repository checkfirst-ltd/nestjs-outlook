# Configuration Reference

Configuration for `MicrosoftOutlookModule`, supplied through `forRoot()` or `forRootAsync()`.

## Registration methods

| Method | Signature | Notes |
|--------|-----------|-------|
| `forRoot` | `MicrosoftOutlookModule.forRoot(config: MicrosoftOutlookConfig)` | Synchronous registration with a static config object. |
| `forRootAsync` | `MicrosoftOutlookModule.forRootAsync({ useFactory, inject, imports })` | Async registration; `useFactory` returns `MicrosoftOutlookConfig` (or a Promise of it). |

Both are generated by `ConfigurableModuleBuilder`. The module registers `ScheduleModule`,
`EventEmitterModule`, and the required TypeORM entities internally.

## `MicrosoftOutlookConfig`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `clientId` | `string` | Yes | — | Microsoft Entra application (client) ID. |
| `clientSecret` | `string` | Yes | — | Microsoft Entra client secret. |
| `redirectPath` | `string` | Yes | — | OAuth callback path, e.g. `auth/microsoft/callback`. |
| `backendBaseUrl` | `string` | Yes | — | Public base URL of the backend, e.g. `https://api.example.com`. Used to build the redirect URI and webhook notification URLs. |
| `basePath` | `string` | No | — | API base path prepended to controller routes, e.g. `api/v1`. |
| `calendarWebhookPath` | `string` | No | `/calendar/webhook` | Path of the calendar webhook endpoint. |
| `state` | `MicrosoftOutlookStateConfig` | No | in-memory | Shared-state backend for locks and rate-limit budgets across containers. |
| `revocationEmitFlagTtlMs` | `number` | No | `604800000` (7 days) | TTL for the dedupe flag that suppresses duplicate `USER_REFRESH_TOKEN_INVALID` emits during webhook bursts. |

## `MicrosoftOutlookStateConfig`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `redis.client` | `RedisLike` | Yes (within `redis`) | — | Host-provided ioredis-compatible client. The module never imports ioredis itself. |
| `redis.keyPrefix` | `string` | No | `"outlook:"` | Prefix applied to all Redis keys. |
| `redis.required` | `boolean` | No | `false` | If `true`, module init throws when the Redis `PING` probe fails (instead of falling back to in-memory). |

When `state.redis.client` is omitted, the lock store and rate-limit store use in-process
in-memory implementations, suitable only for single-container deployments.

## DI tokens (exported constants)

| Constant | Value | Purpose |
|----------|-------|---------|
| `MICROSOFT_CONFIG` | `'MICROSOFT_CONFIG'` | Provider token resolving to the active `MicrosoftOutlookConfig`. |
| `OUTLOOK_LOCK_STORE` | `'OUTLOOK_LOCK_STORE'` | Token for the lock store (Redis or in-memory). |
| `OUTLOOK_RATE_LIMIT_STORE` | `'OUTLOOK_RATE_LIMIT_STORE'` | Token for the rate-limit store (Redis or in-memory). |
| `GRAPH_ERROR_CODES.MAILBOX_NOT_ENABLED` | `'MailboxNotEnabledForRESTAPI'` | Graph error code for an inactive/non-REST mailbox. |
